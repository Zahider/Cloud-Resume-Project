End-to-End RideShare WebApp - JetHop

Welcome to JetHop, an AWS-hosted web application for requesting air rides. Think Uber, but with planes!

Seamlessly integrating ARCGIS maps, JetHop features robust authentication via AWS Cognito User Pools and IAM ensured secure access. Leveraging AWS Amplify and CodeCommit enabled rapid deployment and scaling with minimal configuration. Backend infrastructure, including Lambda, DynamoDB, and API Gateway, facilitated efficient ride request storage and reliable service delivery.

Steps taken to create JetHop:


1. Create code commit repository to store my code

2. Create a new admin user for JetHop usage since it is not best practice to use root user

3. Add "AWSCodeCommitPowerUser" policy to new admin IAM user so I can access CodeCommit

4. Create Git credentials for new IAM user to allow HTTPS connections to CodeCommit

5. Clone repository using CloudShell by entering repository and selecting "Clone HTTPS" and pasting it, then enter your HTTTPS Git Credentials. This is for future use to test out code and make sure everything is working correctly

6. Push premade code into repository

7. Host new web app on AWS Amplify

8. Create User Pool in AWS Cognito to authenticate user accessing the website

9. Update config.js file in CodeCommit for our app to use the User Pool in Cognito

10. Copy authentication token for now as we will use it later on. This page will be created later on.

11. Create a new DynamoDB Table to store ride requests.

12. Create a new Role using IAM with "LambdaBasicExecutionRole", assign specific permission "PutItem" and Add ARN of our DynamoDB Table to the resource.

13. Create a new Lambda function and use the execution Role we created.

14. Enter executable backend code into Lambda code source.

15. Deploy and Run Test code to ensure write functionality into our DynamoDB table.

16. Create a new REST api with edge optimized endpoint for faster service. This is for communication accross the internet but you can also chose private for local or a regional endpoint as well based on your needs. Then create an authorizer to hook up API gateway and cognito together.

17. Pasted AWS Auth token recieved from website earlier into Test authorizer function and confirmed with Code 200

18. Create a resource in your API to connect to the Lambda function. Create Dev stage and paste Invoke URL into CodeCommit config.js file.

19. Now refresh the Web Application and save time by calling yourself a Jet to Hop on!
